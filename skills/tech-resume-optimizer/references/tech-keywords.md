## 前端工程师 (Frontend Engineer)

### 初级 (0-2年)

**核心语言**:

- JavaScript (ES6+)
- HTML5 / CSS3
- TypeScript (基础)

**框架/库**:

- React / Vue.js / Angular (至少精通一个)
- jQuery (了解即可)

**工具链**:

- Webpack / Vite
- npm / yarn
- Git
- Chrome DevTools

**CSS相关**:

- Flexbox / Grid
- Sass / Less
- Bootstrap / Tailwind CSS
- 响应式设计

**其他**:

- RESTful API
- AJAX / Fetch
- 浏览器兼容性
- 性能优化基础

### 中级 (2-5年)

**在初级基础上增加**:

**状态管理**:

- Redux / Vuex / MobX
- Context API
- Zustand / Jotai

**高级框架特性**:

- React Hooks / Vue Composition API
- Next.js / Nuxt.js (SSR)
- React Router / Vue Router

**工程化**:

- Babel
- ESLint / Prettier
- Jest / Cypress (单元测试/E2E测试)
- CI/CD 基础

**性能优化**:

- 代码分割 (Code Splitting)
- 懒加载 (Lazy Loading)
- 虚拟滚动
- 浏览器渲染原理
- Lighthouse 评分优化

**TypeScript**:

- 类型系统深入
- 泛型
- 装饰器

### 高级/资深 (5年+)

**在中级基础上增加**:

**架构设计**:

- 微前端 (Micro Frontends)
- 组件库设计
- 设计模式应用
- 前端监控体系

**构建优化**:

- Webpack 深度配置
- Tree Shaking
- Bundle 分析
- CDN 优化

**跨端开发**:

- React Native / Flutter
- Electron
- Taro / Uni-app

**前沿技术**:

- WebAssembly
- WebGL / Three.js
- PWA (Progressive Web App)
- Service Workers

**团队管理**:

- Code Review
- 技术选型
- mentorship
- 技术方案设计

## ATS优化建议

### 关键词密度

- 核心技能在简历中出现 3-5 次
- 项目描述中每个关键技术出现 1-2 次
- 避免机械重复,自然融入上下文

### 格式要求

- 使用标准章节标题 (Skills, Experience, Projects)
- 避免表格、图片、文本框
- 使用清晰的 bullet points
- PDF 格式确保跨平台兼容

### 常见陷阱

- ❌ "熟悉多种编程语言" → ✅ "精通 Python, 熟练 Java, 了解 Go"
- ❌ "负责系统开发" → ✅ "设计并实现微服务架构,支持日均100万请求"
- ❌ "优化了性能" → ✅ "通过Redis缓存将API响应时间从500ms降至50ms,提升90%"

## 如何使用本参考

1. **确定目标岗位**: 根据用户求职意向选择对应岗位级别
2. **提取关键词**: 从JD(职位描述)中提取关键技能要求
3. **对比分析**: 检查简历是否覆盖必备关键词
4. **补充建议**: 对缺失的关键技能提供学习或展示建议
5. **优化表述**: 将关键词自然融入项目和经历描述

**注意**: 关键词只是基础,更重要的是通过具体项目证明能力。避免简单罗列技术名词,要说明应用场景和成果。
